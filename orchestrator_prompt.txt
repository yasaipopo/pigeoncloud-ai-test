### ループ（最大 $MAX_ROUNDS 回）

#### Step 1: 未実装テストをカウント
以下のPythonコードを bash で実行して未実装spec一覧を取得：

```bash
python3 -c "
import re, yaml
from pathlib import Path

specs_dir = Path('/app/specs')
tests_dir = Path('/app/tests')
missing = {}

priority = ['uncategorized','fields','table-definition','users-permissions',
            'notifications','chart-calendar','system-settings','workflow',
            'layout-ui','auth','comments-logs','csv-export','records',
            'reports','public-form','filters']

for spec in priority:
    yaml_file = specs_dir / f'{spec}.yaml'
    if not yaml_file.exists():
        continue
    data = yaml.safe_load(yaml_file.read_text())
    yaml_cases = [str(c['case_no']) for c in data.get('cases',[]) if c and 'case_no' in c]
    spec_js = tests_dir / f'{spec}.spec.js'
    js_cases = []
    if spec_js.exists():
        js_cases = list(set(re.findall(r\"test(?:\.skip)?\\s*\\(\\s*['\\\`]([\\d]+-[\\d]+(?:-[\\d]+)?|[\\d]+)\", spec_js.read_text())))
    m = [c for c in yaml_cases if c not in js_cases]
    if m:
        missing[spec] = len(m)
        print(f'{spec}: {len(m)}件')

total = sum(missing.values())
print(f'TOTAL: {total}')
"
```

未実装が0件なら終了。前回から減っていなければ終了。

#### Step 2: implement_skips（大規模spec分割対応・Agent toolで並列起動）

まず以下のPythonを実行して、specごとのskipリストとチャンク分割計画を取得する：

```bash
python3 -c "
import re
from pathlib import Path

CHUNK_SIZE = 40  # 1エージェント当たりの最大skip数

tests_dir = Path('/app/tests')
priority = ['uncategorized','fields','table-definition','users-permissions',
            'notifications','chart-calendar','system-settings','workflow',
            'layout-ui','auth','comments-logs','csv-export','records',
            'reports','public-form','filters']

tasks = []
for spec in priority:
    spec_js = tests_dir / f'{spec}.spec.js'
    if not spec_js.exists():
        continue
    content = spec_js.read_text()
    skip_ids = re.findall(r\"test\.skip\s*\(\s*['\\\`]([\\d]+-[\\d]+(?:-[\\d]+)?|[\\d]+)\", content)
    if not skip_ids:
        continue
    if len(skip_ids) <= CHUNK_SIZE:
        tasks.append({'spec': spec, 'target_ids': '', 'count': len(skip_ids)})
    else:
        for i in range(0, len(skip_ids), CHUNK_SIZE):
            chunk = skip_ids[i:i+CHUNK_SIZE]
            tasks.append({'spec': spec, 'target_ids': ','.join(chunk), 'count': len(chunk)})

if not tasks:
    print('TOTAL:0')
else:
    for t in tasks:
        print(f\"{t['spec']}|{t['count']}|{t['target_ids']}\")
    print(f'TOTAL:{sum(t[\"count\"] for t in tasks)}')
"
```

出力の各行（`spec|count|target_ids` 形式）をパースして、
**Agent toolのrun_in_background=trueで最大 $MAX_WORKERS 個まで並列起動**する。

target_ids が空 = そのspecの全skipを担当。
target_ids が設定されている = そのIDのみ担当（大規模spec分割）。

## Step 2: implement_skips を実行

/app/tests/{TARGET_SPEC}.spec.js の test.skip() を実装する。

### 担当IDの確認
TARGET_IDS="{TARGET_IDS}"

TARGET_IDS が空でない場合: そのIDのtest.skipのみ実装する（他には手を付けない）
TARGET_IDS が空の場合: ファイル内の全test.skipを実装する

担当IDの確認方法：
```bash
node -e "
const fs = require('fs');
const content = fs.readFileSync('tests/{TARGET_SPEC}.spec.js', 'utf8');
const targetIds = '{TARGET_IDS}'.split(',').filter(Boolean);
const skipPattern = /test\.skip\s*\(\s*['\`]([\d]+-[\d]+(?:-[\d]+)?|[\d]+)/g;
let m; const found = [];
while ((m = skipPattern.exec(content)) !== null) {
  if (!targetIds.length || targetIds.includes(m[1])) found.push(m[1]);
}
console.log('担当skip数:', found.length, '\nIDs:', found.join(', '));
"
```

### 実装手順
CLAUDE.md の「MODE=implement_skips」セクションの手順に従いつつ:

1. 上記で確認した担当test.skipを1件ずつ実装（最大3回試行）
2. 実装したら npx playwright test tests/{TARGET_SPEC}.spec.js --grep "^{TEST_ID}:" で単体確認
3. 全件実装後に担当分のみ再確認
4. 結果を /app/reports/agent-{AGENT_NUM}/repair_report.md に記録

#### Step 4: repair_specs（大規模spec分割対応・Agent toolで並列起動）

⚠️ **重要: 1 agentが担当するテスト数は最大40件まで。それ以上は必ずシャード分割すること。**

まず全specのテスト数を確認してシャード分割計画を作る：

```bash
python3 -c "
import re
from pathlib import Path

SHARD_SIZE = 40   # 1 agentあたりの最大テスト数（これを超えたらシャード分割）
tests_dir = Path('/app/tests')

tasks = []
for spec_js in sorted(tests_dir.glob('*.spec.js')):
    content = spec_js.read_text()
    total = len(re.findall(r'\btest\s*\(', content))
    if total == 0:
        continue
    spec = spec_js.stem
    if total > SHARD_SIZE:
        shards = min(15, (total + SHARD_SIZE - 1) // SHARD_SIZE)  # 切り上げ、最大15
        for s in range(1, shards + 1):
            tasks.append({'spec': spec, 'total': total, 'shard': s, 'shards': shards})
    else:
        tasks.append({'spec': spec, 'total': total, 'shard': None, 'shards': None})

for t in tasks:
    shard_str = f\"{t['shard']}/{t['shards']}\" if t['shard'] else ''
    print(f\"{t['spec']}|{t['total']}|{shard_str}\")
print(f'TOTAL_TASKS:{len(tasks)}')
"
```

TOTAL_TASKSを確認して、**MAX_WORKERS単位でバッチ処理**する：
- tasks数がMAX_WORKERSを超える場合は、MAX_WORKERS個ずつ順番に起動してdone待ち後に次のバッチを起動
- 全taskが完了するまでバッチを繰り返す

各行（`spec|total|shard/shards` 形式）をパースして起動。
