# モデル使い分けルール

## Opus（高度な判断・設計）
- フローの設計・修正方針決定
- 品質チェック・レビュー
- 新しいパターンの最初の1件を自分で確実に実行→確認
- 問題の根本原因分析

## Sonnet（実行・並列作業）
- エージェント並列タスク（`model: "sonnet"`）
- 確認済みパターンの展開適用
- テスト実行・結果収集

## フロー
```
① Opusで方針決定・最初の1件を実行
② Opusで結果チェック（確実にOK確認）
③ Sonnetエージェントに同じパターンで並列展開
④ Opusで各エージェント結果をチェック
```

## 禁止
- ❌ Opusで大量の単純作業（トークン浪費）
- ❌ Sonnetでフロー設計や方針変更
- ❌ チェックなしでいきなり並列展開
