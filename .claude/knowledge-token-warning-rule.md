# トークン数警告ルール

## ルール
会話が長くなりすぎたら（以下の兆候が出たら）ユーザーに警告する:

1. **コンテキスト圧縮が発生した時** → 「コンテキストが圧縮されました。新しい会話で続けた方が効率的です」
2. **同じファイルを3回以上読み直している時** → 会話が長すぎるサイン
3. **エージェントの結果を待つ間に何度も空のやり取りが発生** → 別会話で管理すべき
4. **1つの会話で2時間以上経過** → 区切りを提案

## 対策
- current-task.mdに状態を記録して、新しい会話で引き継ぐ
- 重い作業（テスト実行、spec修正）はエージェントに任せ、会話自体は軽く保つ
- 「待ち」の時は会話を区切って、完了後に新しい会話で確認
