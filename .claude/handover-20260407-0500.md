## 注意事項・懸念点

### sync-results の仕組み
- テスト結果は `runs/cases` テーブルに保存される
- `POST /pipeline/sync-results` で pipeline テーブルの `stagingResult` に反映
- **テストタイトル → movie ID → pipeline caseNo のマッピング**で同期
- movie 形式（AT01, RC01等）は pipeline の `movie` フィールドでマッチ

### 動画パスの3段階フォールバック
1. `runs/{runId}/{caseId}/video/` — runs テーブルの videoKey
2. pipeline テーブルの videoKey
3. `videos/{spec}/{movie}/video.webm` — 旧パス（手動アップロード分）

### ステップスクショの命名規則
- `steps/{spec}/{movie}/{stepId}-s{N}.jpg` — N はフロー内の行番号
- **✅ がある行は全て撮る**（同画面でも）
- 撮り忘れると sheet.html の📷が壊れた画像を表示する

### 無効な Angular ルート（修正済みだが知見として重要）
- `/admin/dataset__N/setting/*` → `/admin/dataset/edit/N`
- `/admin/user` → `/admin/admin`
- `/admin/organization` → 現在のUIに存在しない
- `/admin/step_mail` → 現在のUIに存在しない
- `/admin/webhook` → 現在のUIに存在しない

### beforeEach の正しいパターン
```javascript
test.beforeEach(async ({ page }) => {
    await page.context().clearCookies();
    await page.goto(BASE_URL + '/admin/login', { waitUntil: 'domcontentloaded', timeout: 15000 }).catch(() => {});
    await page.evaluate(() => { try { localStorage.clear(); sessionStorage.clear(); } catch {} });
    if (!page.url().includes('/login')) {
        await page.goto(BASE_URL + '/admin/login', { waitUntil: 'domcontentloaded', timeout: 15000 }).catch(() => {});
    }
    if (page.url().includes('/login')) {
        await page.fill('#id', EMAIL, { timeout: 15000 }).catch(() => {});
        await page.fill('#password', PASSWORD, { timeout: 15000 }).catch(() => {});
        await page.locator('button[type=submit].btn-primary').first().click({ timeout: 15000 }).catch(() => {});
        await page.waitForSelector('.navbar', { timeout: 15000 }).catch(() => {});
    }
    await waitForAngular(page);
    await page.waitForSelector('.sidebar-nav a, .nav-link', { timeout: 10000 }).catch(() => {});
    await closeTemplateModal(page);
});
```
