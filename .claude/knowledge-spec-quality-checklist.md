# spec品質チェックリスト（全spec共通）

最終更新: 2026-04-10

各specの品質仕上げ時に、以下を全項目クリアすること。

---

## 1. テスト観点の完全性

- [ ] **各ケースのタイトルに対して、結果を直接確認する✅が最低1つある**
  - 「〜を追加できること」→ 追加されたものが画面に表示されていることの✅
  - 「〜が削除されること」→ 対象が消えていることの✅
  - 「エラーなし」だけでは不十分
- [ ] **操作が`if`条件でスキップされていない**
  - `if (menuVisible)` `if (count > 0)` 等で操作をスキップ → テストが通るが何も検証していない
  - 操作できないならエラー（`expect().toBeVisible()` / `throw`）にする
- [ ] **yamlのフローとspec.jsの実装が1対1で対応している**
  - yamlに書いてある操作手順が、spec.jsで全て実行されている
  - spec.jsにあるがyamlに書いていないステップがない

## 2. データ表示の完了待ち

- [ ] **スケルトン/ローディング状態でアサーション・スクショを撮っていない**
  - テーブル表示: `waitForSelector('tbody tr td')` でデータ行が出るまで待つ
  - ウィジェット: スケルトン（グレーのプレースホルダー）が消えるまで待つ
  - グラフ/集計: 実際の数値やグラフ描画が完了するまで待つ
- [ ] **「データはありません」状態をOKとしていない**
  - データ投入後に「データはありません」表示 → テストデータの投入タイミングか待ち時間の問題
- [ ] **成功トーストが表示される操作では、トースト表示を待ってからスクショ**
  - 保存/追加/削除後に表示される緑のトースト → 操作成功の証拠として撮る

## 3. スクショの品質

- [ ] **各✅行のスクショが、その✅の内容を証明する画面状態になっている**
  - 「ウィジェットが追加されていること」のスクショ → ウィジェットの中身が見えている
  - 「メニューが存在しないこと」のスクショ → メニューが開いた状態で撮る
- [ ] **autoScreenshotの呼び出しが適切なタイミングにある**
  - 1ケースで複数の画面状態を確認する場合、最も重要な状態でスクショ
  - 必要なら手動で追加スクショ（autoScreenshotに加えて`page.screenshot`）
- [ ] **ローディング完了後にスクショを撮っている**
  - `waitForAngular` + `waitForLoadState('networkidle')` 後にスクショ

## 4. detailedFlowの記述品質

- [ ] **CSSセレクター・Playwrightコードが一切含まれていない**
  - `.navbar`, `[role=tab]`, `.alert-danger` → 人間向けの日本語表現
  - `expect(...)`, `page.locator(...)` → 完全削除
  - 詳細は `knowledge-detailed-flow-writing-rule.md` 参照
- [ ] **番号がケース内番号（30-1, 30-2...）になっている**
  - 通し番号（1, 2, 3...23）は禁止
- [ ] **タイムスタンプが最新テスト実行の実測値になっている**
  - 古い値のまま放置しない。テスト再実行後に必ず更新

## 5. 動画との同期

- [ ] **各ケースのタイムスタンプと実際の動画が概ね一致している**
  - ±3秒程度のズレは許容。それ以上ズレていたらタイムスタンプ更新
- [ ] **テスト実行後にアップロード→sync-results→yaml同期の全手順を実行している**
  - スクショアップロード
  - 結果アップロード（RUN_ID取得）
  - sync-results（stagingRunId紐付け）
  - upload_yaml_specs --overwrite（detailedFlow更新）

## 6. エラーハンドリング

- [ ] **メニュー・ダイアログの表示確認がexpectで必須化されている**
  - `if (menuVisible)` → `await expect(menu).toBeVisible()`
  - ダイアログが開かない = テスト失敗にする
- [ ] **UIが見つからない場合にgraceful skipしていない**
  - `if (!element) return;` → `await expect(element).toBeVisible()`
  - 「見つからなかったのでスキップ」は禁止

---

## 適用手順（各specで実施）

1. yamlのdetailedFlowを読み、spec.jsと突き合わせ
2. 上記チェックリスト全項目を確認
3. 問題点を洗い出して一括修正
4. テスト実行 → 全passed確認
5. スクショを目視確認（ローディング状態でないか）
6. タイムスタンプ更新 → アップロード → sheet.html確認
