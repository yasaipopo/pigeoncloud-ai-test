# spec品質チェックリスト（全spec共通）

最終更新: 2026-04-10

各specの品質仕上げ時に、以下を**全項目**クリアすること。
1項目でもNGがあればそのspecは未完了。

---

## A. yamlとspec.jsの整合性

- [ ] yamlのdetailedFlowに書かれた全ステップ（XX-1, XX-2...）がspec.jsに実装されている
- [ ] spec.jsに実装されている操作で、yamlに記載されていないものがない
- [ ] yamlの各ケースタイトル（「〜を追加できること」等）に対して、そのタイトルの結果を直接確認する✅がyaml・spec.jsの両方にある
  - 例: タイトル「ビューコンテンツを追加できること」→「ビューコンテンツが画面に表示されていること」の✅が必須
  - 例: タイトル「タブを削除できること」→「タブが消えていること」の✅が必須
  - 「エラーがないこと」だけでは不十分（エラーがないことと成功したことは別）
- [ ] spec.jsのコメント番号がyamlのケース内番号と一致している（`// 30-1.`形式）
- [ ] spec.jsのコメント文言がyamlの日本語テキストと一致している

## B. データ表示の完了待ち

- [ ] ページ遷移後に`waitForAngular(page)`だけでなく、**表示を期待する具体的な要素**が出るまで待っている
  - テーブル表示: `waitForSelector('tbody tr td', { timeout: 15000 })` でデータセルが出るまで待つ
  - ウィジェット: スケルトン（グレーのプレースホルダー行）が消えて実データが出るまで待つ
  - 例: `await page.waitForFunction(() => !document.querySelector('.skeleton, .placeholder'))`
- [ ] 保存/追加/削除操作の後に、**結果が画面に反映されるまで**待っている
  - ダイアログが閉じるのを待つ: `waitFor({ state: 'hidden' })`
  - 新しい要素が出るのを待つ: `waitForSelector` または `expect().toBeVisible()`
  - 成功トースト（緑の通知）が出るのを待つ: `waitForSelector('.toast-success, [class*="success"]')`
- [ ] 「データはありません」表示の状態でアサーションをOKにしていない
- [ ] テーブルのスケルトン行（グレーのプレースホルダー）が表示されている状態でアサーションをOKにしていない
- [ ] APIレスポンス待ちが必要な箇所で`waitForTimeout`の代わりに`waitForResponse`または`waitForLoadState('networkidle')`を使っている

## C. アサーション（✅の確認）の品質

- [ ] 「〜が表示されていること」の確認で、**要素の存在だけでなく中身**も確認している
  - テーブルウィジェット: ヘッダーだけでなくデータ行（`tbody tr td`）が1行以上ある
  - 集計ウィジェット: 実際の数値またはグラフが描画されている
  - 掲示板: テキストコンテンツが表示されている
- [ ] 「〜が追加されていること」の確認で、**追加前後の差分**または**追加された要素の特定**ができている
  - 操作前のcount → 操作後のcount > 操作前、または
  - 追加されたアイテムの名前/テキストが画面上に存在する
- [ ] 「エラーがないこと」の確認で、以下の**両方**をチェックしている
  - `.alert-danger`が0件
  - bodyテキストに`Internal Server Error`が含まれない
- [ ] 「〜が存在しないこと」の確認で、`toHaveCount(0)`または`not.toBeVisible()`を使っている
  - 「メニューに削除がないこと」→ メニューを実際に開いた状態で`toHaveCount(0)`
- [ ] `if`条件でアサーションをスキップしている箇所がない
  - `if (menuVisible) { expect... }` → `await expect(menu).toBeVisible(); expect...`
  - UIが見つからない = テスト失敗にする

## D. スクショの品質

- [ ] 各✅行のスクショが、**その✅が主張する内容を視覚的に証明**している
  - 「ウィジェットが追加されていること」→ ウィジェット内にデータが表示されている画面
  - 「タブが消えていること」→ タブ一覧に対象タブがない画面
  - 「メニューに削除がないこと」→ メニューが**開いた状態**でスクショ
- [ ] スクショにローディング状態（スケルトン行、スピナー、「読み込み中」）が写っていない
- [ ] autoScreenshotの呼び出しが、そのケースの**最後のアサーション完了後**にある
- [ ] 1ケースで複数の画面状態を確認するケースで、最も重要な確認結果がスクショに撮れている
  - 例: dash-060は「掲示板表示確認」と「メニューに削除がない確認」の2つがあるが、後者が重要

## E. detailedFlowの記述

- [ ] CSSセレクターが一切含まれていない（`.navbar`, `.alert-danger`, `[role=tab]`, `.modal.show`等）
- [ ] Playwrightコードが一切含まれていない（`expect(`, `page.locator(`, `waitFor`等）
- [ ] URLパスが一切含まれていない（`/admin/dashboard` → 「ダッシュボード画面」）
- [ ] ボタン・メニュー名は「」で囲んでいる
- [ ] 画面名は「○○画面」と書いている
- [ ] 「モーダル」ではなく「ダイアログ」と書いている
- [ ] ステップ番号がケース内番号（30-1, 30-2...）になっている（通し番号1,2,3...は禁止）
- [ ] タイムスタンプ[M:SS]がそのspecの最新テスト実行の実測値になっている

## F. エラーハンドリング

- [ ] 操作対象のUI要素（ボタン、メニュー、ダイアログ）が`expect().toBeVisible()`で表示確認されてからクリックしている
- [ ] `if (await element.isVisible())` で操作をスキップしている箇所がない → `expect`に変更
- [ ] `.catch(() => {})` でエラーを握りつぶしている箇所が、**本当に握りつぶして良い箇所のみ**に限定されている
  - OKな例: `closeTutorialModal`（チュートリアルが出ない場合もある）
  - NGな例: テスト対象の操作（ボタンクリック、フォーム入力）の`.catch(() => {})`
- [ ] メニューが開かなかった場合、ダイアログが表示されなかった場合はテスト失敗になる

## G. 動画との同期・アップロード

- [ ] spec.jsを修正した場合、テストを再実行して最新の動画を撮影している
- [ ] STEP_TIMEログからyamlのタイムスタンプを更新している
- [ ] アップロード全手順を実行している
  1. `upload_step_screenshots.py`（スクショ→S3）
  2. `upload_results.py`（結果＋動画→S3、RUN_ID取得）
  3. `sync-results`（pipeline DBにrunId紐付け）
  4. `upload_yaml_specs.py --overwrite`（yaml内容更新）
- [ ] `generate-screenshot-map.py`を再実行してscreenshot-map.jsonを更新している
- [ ] sheet.htmlで動画を再生し、フローのハイライトと動画内容が概ね一致している（±3秒以内）

## H. spec.jsのコード品質

- [ ] テスト対象の操作が`if`分岐内に入っていない（操作が確実に実行される構造になっている）
- [ ] `waitForTimeout`の使用が最小限で、それぞれに理由がある
  - OKな例: Angular描画待ち（`waitForAngular`では不十分な場合のフォールバック）
  - NGな例: 「なんとなく1秒待つ」
- [ ] 同じケース内で同名の変数が再宣言されていない（`const errorAlert`が複数回→名前を変える）
- [ ] `console.log`にケース番号が含まれている（デバッグ時にどのケースか特定できる）

---

## 適用手順

1. yamlを読み、spec.jsと1行ずつ突き合わせてA〜Hの全項目を確認
2. 問題リストを一括で洗い出す（1つ直して再確認ではなく、全部洗い出してから一括修正）
3. spec.jsとyamlを修正
4. テスト実行 → 全passed確認
5. **スクショを1枚ずつ目視確認**（ローディング状態でないか、主張と画面が一致しているか）
6. タイムスタンプ更新 → アップロード → sheet.htmlで動画再生確認
