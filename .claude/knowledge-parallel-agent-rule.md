# 並列エージェント実行ルール

## ルール: 「1-2件テスト → チェック → 並列展開」

並列エージェントでタスクを実行する場合、以下のフローを**必ず**守ること:

### フロー
```
① まず1-2件を自分で実行（またはagent 1つで実行）
② 結果をチェック（品質・正確性・副作用）
③ 問題なければ、同じ指示で並列agentに展開（最大5並列）
④ 各agent完了後にチェック
```

### 禁止事項
- ❌ いきなり5-10並列で投げる（品質不明のまま大量実行）
- ❌ チェックせずに次のバッチを投入する
- ❌ agentの完了報告だけ見て中身を確認しない

### チェック内容
- 全件処理されたか（漏れなし）
- 内容が正しいか（ランダムサンプリング）
- 副作用がないか（他ファイルを壊していない）
- フォーマットが統一されているか

### 適用場面
- detailed_flow作成
- spec.js修正
- test.step化
- その他、同じパターンの作業を複数ファイルに適用する場合
