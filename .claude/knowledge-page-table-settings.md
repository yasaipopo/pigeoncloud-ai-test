# テーブル設定ページ知見

## URL
- テーブル設定: `/admin/dataset/edit/{tableId}`
- 新規テーブル作成: `/admin/dataset/edit/new`

## テーブル作成フロー
1. サイドバーの+ボタン → 「テーブル追加」メニュー → 「テーブル追加画面へ」
2. または直接 `/admin/dataset/edit/new` に遷移
3. テーブル名入力: `input.form-control:visible` の最初のもの（非表示の重複inputあり注意）
4. 「＋項目を追加する」ボタン → フィールドタイプ選択モーダル
5. フィールドタイプクリック → **設定モーダル（.settingModal）が自動で開く**
6. 設定モーダルを閉じてから次のフィールド追加
7. `button[type=submit].btn-primary` の `.last()` で保存

## 設定モーダル（.settingModal）
- フィールド追加後に自動で開く
- 「変更する」ボタンで保存して閉じる
- Escapeで閉じるとAngularの状態が不安定になることがある
- **リロードするとフォーム入力がリセットされるため不可**
- JS DOM操作（classList.remove('show')）は`.settingModal`のみに限定（`.modal.show`を全部消すと他のモーダルも消える）

## フィールドタイプ選択モーダル
- セレクター: `.modal.show` 内の `text=日時` 等
- タイプ一覧: 自動採番、文字列(一行)、文章(複数行)、数値、Yes/No、選択肢(単一選択)、選択肢(複数選択)、日時、画像、ファイル、他テーブル参照、計算、関連レコード一覧、固定テキスト

## .pc-field-block
- フィールド一覧の各フィールド要素
- システムフィールド（全ユーザー等）はCSS非表示 → `.first()` で取ると非表示要素
- `:visible` フィルタか `.overSetting` の存在で判定

## テーブル設定タブ
- 基本設定、メニュー、一覧画面、詳細・編集画面、CSV、ワークフロー、地図設定、その他
- フィールド一覧は基本設定タブの下方（スクロール）

## UI文言
- 保存ボタン: 「変更する」（旧: 「更新する」）
- フィールド設定モーダルヘッダー: 「項目編集」
