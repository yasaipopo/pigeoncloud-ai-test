# detailedFlow 記述ルール（テスター向け）

最終更新: 2026-04-10

## 大原則

detailedFlowは**人間のテスターが動画を見ながら読むもの**。
CSSセレクター・Playwrightコード・技術的表現は一切使わない。

---

## 禁止表現 → 正しい表現

### CSSセレクター系

| ❌ 禁止 | ✅ 正しい書き方 |
|---|---|
| `.navbar が表示されていること` | `ページ上部のナビゲーションメニューが表示されていること` または `ページが正常に表示されていること` |
| `.alert-danger が表示されないこと` | `エラーメッセージが表示されないこと` |
| `[role=tab] "HOME"` | `「HOME」タブ` |
| `[role=tabpanel]` | `タブの内容` |
| `[role=tablist]` | `タブ一覧` |
| `[role=combobox]` | `選択ボックス` / `ドロップダウン` |
| `[role=menu]` / `[role=menuitem]` | `メニュー` / `メニュー項目` |
| `.modal.show` | `ダイアログ` |
| `.overSetting` | `設定画面` |
| `.pc-field-block` | `フィールド設定ブロック` |
| `.sidebar-nav` | `左メニュー` |
| `aside内に` | `サイドパネルに` |
| `.fa-plus` / `fa-search` | `＋アイコン` / `虫眼鏡アイコン` |
| `.btn-primary` / `.btn-danger` | `送信ボタン` / `削除ボタン`（文脈に応じて） |
| `.toast-success` | `成功メッセージ（トースト）` |
| `.toast-error` | `エラーメッセージ（トースト）` |
| `data-ng-ready` / `ng-reflect` | 使わない（内部実装の話） |
| `.cdk-drag` | `ドラッグ可能な項目` |

### URL・パス系

| ❌ 禁止 | ✅ 正しい書き方 |
|---|---|
| `/admin/dashboard に遷移` | `ダッシュボード画面に遷移` |
| `/admin/dataset` | `テーブル管理画面` |
| `/admin/notification` | `通知設定画面` |
| `/admin/logs` | `ログ画面` |
| `/admin/csv` | `CSV履歴画面` |
| `/admin/rpa` | `コネクト管理画面` |
| `URLが /admin/xxx を含むこと` | `○○画面に遷移していること` |

### Playwrightコード（完全削除）

以下のような行はdetailedFlowに**絶対に書かない**:
- `expect(page.locator('.navbar')).toBeVisible()`
- `await page.click('button.btn-primary')`
- `page.locator('selector').fill('value')`
- `waitForSelector('.xxx', { timeout: N })`
- `selector: .cdk-drag.field-drag`
- `- await expect(page.locator(...))`

### UI用語統一

| 用語 | 意味 |
|---|---|
| ダイアログ | ポップアップ表示されるフォームや確認画面（「モーダル」は技術用語なので使わない） |
| ドロップダウン | クリックすると選択肢が出るUI |
| タブ | 画面上部の切り替えボタン |
| トースト | 画面端に一時的に表示される通知メッセージ |
| サイドパネル | 画面右側に出てくるパネル（コメント表示等） |
| ナビゲーションメニュー | ページ上部に表示されるメニューバー |
| 左メニュー | 画面左側の固定メニュー |

## 書き方の基本ルール

1. **テスターが動画を見ながら「ここを確認して」と言われてわかる表現**にする
2. 操作手順は「〜をクリック」「〜を入力して」のように動詞で終わる
3. 確認ポイント（✅/🔴）は「〜が表示されていること」「〜であること」のように名詞で終わる
4. ボタンやメニュー名は「」（かぎ括弧）で囲む
5. テーブル名・フィールド名など固有名詞も「」で囲む
6. 画面名は「○○画面」と書く（URLパスは使わない）

## 手本: dashboard.yaml（DB01）

```
[0:00] dash-010 ── ダッシュボード画面が正常に表示されること
1. ログイン後、ダッシュボード画面に遷移する
2. ✅ ページ上部のナビゲーションメニューが表示されていること
3. ✅ 「HOME」タブが表示されていること
4. ✅ ダッシュボード画面に遷移していること

[0:13] dash-020 ── 新しいダッシュボードタブを作成できること
5. タブ一覧右側の「+」ボタンをクリック
6. ダッシュボード作成ダイアログで名前を入力して送信
7. ✅ 新しいタブがタブ一覧に表示されること
```

## 適用手順（各specで実施）

1. specs/*.yaml の detailed_flow を上記ルールで書き換え
2. `python3 scripts/generate-screenshot-map.py` で screenshot-map.json 再生成
3. テスト実行 → アップロード → sheet.html で確認
